# X2MD V4 README 重构设计

## 目标

把仓库首页从偏技术说明书重构为面向普通用户的产品介绍页。读者应在一分钟内理解：

1. X2MD 是什么。
2. 它能保存哪些网站的内容。
3. 保存后的 Markdown 有什么价值。
4. 如何在 Mac 上安装并保存第一篇内容。

README 前 80% 面向非技术用户，末尾保留简短的开发、构建和详细文档入口。

## 项目简介

> X2MD 是一个本地优先的网页收藏工具。点击一次，就能把 X、LINUX DO、飞书和微信公众号内容，完整保存为可编辑、可搜索的 Markdown，并直接放进 Obsidian 或你的本地资料库。

该简介同时作为 GitHub 仓库 About 描述的推荐文案；如字符数受限，缩短为：

> 一键把 X、LINUX DO、飞书和微信公众号内容保存为本地 Markdown。

## 视觉方向

采用已确认的 **A：清爽产品型**：

- 白色与浅蓝为主，使用深色文字建立清晰层级。
- 首屏先呈现产品价值，不使用 GraphQL、MV3、API 等技术词汇。
- 图形保持扁平、简洁，适配 GitHub 浅色和深色背景。
- 所有 README 配图使用仓库内 SVG，不依赖外部图床。
- 图片包含可读的替代文本，文字内容不只存在于图片中。

## README 信息结构

### 1. 首屏

- 产品图标、`X2MD` 名称和“看到好内容，一键存进 Obsidian”口号。
- 一句话产品说明。
- Mac App、Chrome 扩展两个主要下载入口。
- 最新版本、构建状态与 License 徽章。

### 2. 一张图说明工作方式

以四步流程展示：发现网页内容 → 点击保存 → X2MD 自动整理 → 本地 Markdown。

### 3. 保存前后对比

- 保存前：网页收藏夹中只有链接，容易失效且难以整理。
- 保存后：本地笔记包含标题、正文、作者、原链接、图片与视频引用，可搜索、编辑和二次创作。

### 4. 支持平台

用简短表格或卡片介绍 X、LINUX DO、飞书和微信公众号。每项只说明用户能保存什么，不展开实现原理。

### 5. 三步安装

1. 下载并打开 Mac App。
2. 安装 Chrome 扩展，并用 App 中的 6 位配对码连接。
3. 选择保存目录，在支持的网站点击书签或保存按钮。

Windows 明确标记为 Beta，并链接到平台能力说明，避免暗示它与 Mac Stable 完全一致。

### 6. V4 核心能力

- 本地优先保存。
- 完整保留正文、链接和媒体。
- 重复内容支持跳过、更新或另存。
- 批量任务支持暂停、继续与失败重试。
- App 统一管理设置、历史和任务。

### 7. 常见问题

只保留普通用户最常见的四类问题：App 与扩展连接、保存目录、重复保存、批量任务。详细配置和故障排查改为链接到独立文档。

### 8. 给开发者

README 末尾保留：

- `bun install`、`bun run dev`。
- `npm run check`、`bun run build:mac`。
- `BUILD.md`、平台支持、V4 迁移说明和 PRD 入口。
- 精简后的核心项目结构。

## 内容边界

- 不在 README 中展示完整配置 JSON。
- 不承诺尚未发布的平台能力。
- 不把 Windows Beta 描述为 Stable。
- 不写无法由当前 V4 运行时或 Release 产物支持的功能。
- 不改动应用、扩展或发布逻辑。

## 验证标准

- README 中所有相对链接与本地图片路径存在。
- GitHub Markdown 标题层级、表格与折叠区块结构有效。
- SVG 可由浏览器打开，文本不截断，浅色和深色背景下均可辨认。
- README 前半部分不要求读者理解开发术语。
- 安装步骤与 V4 Release、Mac Stable 和 Windows Beta 的实际状态一致。
- 文档变更通过仓库隐私和禁止文件检查。
